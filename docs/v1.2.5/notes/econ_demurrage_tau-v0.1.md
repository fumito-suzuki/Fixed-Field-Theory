# FFT-Econ Note v0.2  
## 減価通貨 = τ への課税モデル

本ノートでは、減価通貨（デマレッジ通貨）を  
**「τ（phase-time）に対する課税」**として再解釈する FFT 的モデルをまとめる。

目的：

- 減価を「お金そのものへの罰金」ではなく  
  **「時間を独占することへの料金」**として定式化すること。
- χ（投資ベクトル）の選び方と、ρ_T（基盤の厚み）との整合性を見ること。

---

## 1. 基本アイデア：お金が「時間」を占有している

FFT では、任意のプロジェクト / 行為 P について

- 価値：`V(P)`（観測価値・C-wave の総強度）
- 時間：`T(P) ≃ Δτ_P`（その価値が有効な τ 幅）

を用い、真価を

`S(P) = V(P) / T(P)`

と定義する（別ノート参照）。

ここで **通貨 M 自体が τ を占有している**と考える：

- お金を「寝かせる」 =  
  何も生まない χ に τ を割り当てている状態
- 減価通貨 =  
  「何も生まない τ の占有に課税する仕組み」

---

## 2. ミクロモデル：1 枚の通貨に対する τ 課税

### 2.1 通貨トークンと保持時間

ある通貨トークン \(m_i\) について、

- 額面：`M_i`
- 保持開始からの phase-time：`τ_i`

とする。

**減価率 μ** を導入し、  
「保持しているだけで価値が減る」を

`M_i(τ_i) = M_i(0) · e^{-μ τ_i}`

で表す。

- μ = 0 → 通常通貨（減価なし）
- μ > 0 → 減価通貨（τ に課税）

### 2.2 「税」としての解釈

通貨の保持による「損失」は

`Tax_i(τ_i) = M_i(0) - M_i(τ_i)`

となる。  
これは

> 「この τ の間、そのお金を  
>  何にも使わずに寝かせた料金」

として解釈できる。

---

## 3. マクロモデル：通貨密度 ρ_M の時間発展

空間 R 上に通貨密度 ρ_M(x,t) を導入する。

### 3.1 減価項を含む連続の式

最も単純な形として：

`∂_t ρ_M + ∇·J_M = -μ ρ_M`

と書ける。

- `J_M`：お金の流れ（支払い・投資）
- `-μ ρ_M`：  
  何にも用いられず「停滞しているお金」の減価

ここで μ は

- 高いほど「お金を寝かせにくい」
- 低いほど「貯め込みが許容される」

という、**時間占有に対する課税率**になる。

---

## 4. FFT との接続：τ 課税が χ の選び方を変える

### 4.1 χ の自由度と減価

FFT の自由意志モデルでは、

`FreeWillBandwidth ∝ |dχ/dt|`

が大きいほど「別の χ を選び直せる自由」がある。

経済に適用すると：

- お金を寝かせる（投資を先送りする） =  
  χ を変えずに時間だけ消費している状態。
- 減価通貨で μ を導入する =  
  「χ を変えずに τ を浪費すること」にペナルティをかける。

結果として：

- **短い τ で ρ_T を増やす**ようなプロジェクトへ  
  χ が向きやすくなる。
- 逆に、**長い τ を占有するだけの投機的 χ**は  
  自然に抑制される。

### 4.2 ρ_T 方程式への影響イメージ

元の T-wave 密度方程式：

`∂_t ρ_T = β_1‖∇ψ‖² + β_2‖∇C‖² - γ ρ_T`

ここに「通貨の減価」が連動するモデルとして、

- 減価通貨で生じた Tax を  
  ρ_T を増やす投資に再分配する、
- あるいは γ を下げる方向の投資（環境修復など）に当てる、

とすると、

- **お金を寝かせるほど ρ_T を削り、  
  減価で徴収された分を ρ_T を増やす χ へ戻す**

という「τ 課税 → 基盤回復」ループが作れる。

---

## 5. 真価 S に対する補正：S_eff の導入

真価 S = V/T は、  
「プロジェクトごとの効率」だった。

減価通貨の世界では、  
**通貨を寝かせている間に S が劣化する**ようにするため、  
有効真価 S_eff を導入する。

あるプロジェクト P への投資開始が τ_start、  
実行が τ_exec のとき、

- 待機時間：`Δτ_wait = τ_exec - τ_start`

として、S_eff を

`S_eff(P) = S(P) · e^{-μ Δτ_wait}`

と定義する。

- すぐに実行される投資（Δτ_wait ≈ 0）：  
  S_eff ≈ S（真価がそのまま活きる）
- いつまでも先送りされる投資（Δτ_wait 大）：  
  S_eff が指数的に劣化し、  
  「やる価値」がなくなっていく

→ **「やるなら早くやる。やらないなら他に回す」**  
という χ の選び方を自然に促す。

## 6. TRANSPIT / 螺旋型経済への適用メモ

- TRANSPIT 型のビジネス：  
  「小さな Δτ のループで ρ_T を増やし続ける」典型例。  
  減価通貨と相性が良い。
- 大型・長期投資：  
  減価通貨下でも正当化するには、  
  ρ_T の増加量が非常に大きいか、  
  τ を段階的に区切る設計（フェーズ分割）が必要。

螺旋型経済の視点では：

- **良い χ**：  
  減価で徴収される Tax を上回る速度で ρ_T と V を増やすループを作る。
- **悪い χ**：  
  長い Δτ_wait で何も生まないまま減価させるだけの停滞 χ。

## 8. まとめメモ

- 減価通貨は「お金」への罰則ではなく、  
  **「何もしないまま時間を独占すること」への課税**として定義できる。
- FFT 的には、これは **τ への課税**であり、  
  「χ を変えないまま時間だけ消費する」行為を抑制する。
- この仕組みは、螺旋型経済（基盤 ρ_T を削らずに S を更新し続ける経済）を  
  自然に選好する圧力として働く可能性がある。
